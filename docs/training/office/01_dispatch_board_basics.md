# Dispatch Board Basics

## What you'll learn
- What the dispatch board shows in Kanban and List view
- What each status column means in daily operations
- How to use filters fast during a busy shift
- How to drill into a load detail without losing your place

## Before you start
- You need office access to Dispatch
- You should know basic load lifecycle terms
- Confirm the correct operating company is selected

## Step-by-step
1. Open the Dispatch module.
   [screenshot: Dispatch module selected in main navigation]
2. Start in **Board (Kanban)**.
   [screenshot: Kanban board with status columns]
3. Review the columns:
   - Pending Assignment
   - Assigned
   - In Transit
   - Delivered
   - Completed
   - Cancelled
4. Switch to **Board (List)** when you need table sorting and denser scanning.
   [screenshot: List view with sortable columns]
5. Apply filters in this order:
   - Operating Company
   - Customer
   - Driver
   - Date range
   [screenshot: Filter bar with company/customer/driver/date]
6. Click a load card or row to open detail.
   [screenshot: Load detail drawer open]
7. Verify customer, route, status, driver assignment, and notes before acting.
   [screenshot: Load detail with key fields highlighted]
8. Close detail and continue from the same filtered view.

## Common questions
**Q: When do I use Kanban vs List?**  
A: Use Kanban for stage movement and queue awareness. Use List for sorting and detail scanning.

**Q: Why is a load Delivered but not Completed?**  
A: Delivered means physical delivery happened. Completed usually requires final docs or closeout actions.

**Q: I cannot find a load I just booked. Where do I check first?**  
A: Check Operating Company and date range first, then clear customer/driver filters.

## What to do if something breaks
- If results look wrong, refresh once and reapply filters.
- If a load shows in the wrong stage, capture load ID and escalate to dispatch lead.
- If access is denied, ask supervisor to confirm your role permissions.
- If board does not load, send error screenshot, user email, and timestamp to support.
