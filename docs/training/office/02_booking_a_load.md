# Booking a Load

## What you'll learn
- How to open the v3 booking modal from Dispatch
- What to fill in Tab 1, Tab 2, and Tab 3
- How to add multi-stop pickups and deliveries
- How to verify the load and driver bill were created

## Before you start
- You need permission to book loads
- Confirm customer profile is active and billing-ready
- Have appointment windows and route details ready
- Confirm driver and unit availability

## Step-by-step
1. In Dispatch, click **+ Book Load**.
   [screenshot: + Book Load button in Dispatch header]
2. In Tab 1 (**Load details**), fill:
   - Customer
   - PO#
   - Pickup and delivery details
   - Time windows
   - Hazmat flag if needed
   - Driver instructions
   [screenshot: Tab 1 Load details completed]
3. For multi-stop loads, use:
   - **+ Add pickup**
   - **+ Add delivery**
   [screenshot: Multi-stop controls in modal]
4. Open Tab 2 (**Driver pay**) and fill:
   - Driver assignment
   - Unit
   - Tarps/Lumpers items if applicable
   [screenshot: Tab 2 Driver pay completed]
5. Open Tab 3 (**Customer invoice**) and fill:
   - Rate
   - Detention
   - Free time
   [screenshot: Tab 3 Customer invoice completed]
6. Review all tabs and click Submit.
   [screenshot: Final submit review state]
7. Confirm load appears in dispatch board.
   [screenshot: Newly created load visible on board]
8. Confirm driver bill auto-created for that load.
   [screenshot: Driver bill linked to new load]

## Common questions
**Q: What if I only have partial stop information?**  
A: Enter confirmed details first and update quickly when final times are provided.

**Q: Do detention and free time always matter?**  
A: Yes for billing accuracy. Missing these can create avoidable disputes.

**Q: Submission failed after I filled everything. Why?**  
A: Check validation banners, role gates, or assignment constraints in the modal.

## What to do if something breaks
- Retry once and copy the exact error text.
- If load is created but driver bill is missing, escalate immediately with load ID.
- If modal freezes, reopen and retry, then capture screenshots for support.
- For repeated failures, report timestamp, user, and exact step where it fails.
