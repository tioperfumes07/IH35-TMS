# Driver onboarding checklist (English)

Use this list during the first live week. Supervisors should shadow drivers until every box is checked without assistance.

## Before day one

1. **Phone ready**: charged, mobile data working, WhatsApp/SMS allowed for OTP if your fleet uses phone login.  
2. **PWA installed**: follow [01_pwa_install_guide_en.md](./01_pwa_install_guide_en.md) — icon on home screen, notifications allowed if prompted.  
3. **Credentials**: complete invite / phone / email verification with dispatch; confirm you can open the app offline shell (cached assets) even when signal dips.

## Day one — loads and paperwork

4. **Find assigned loads**: practice filters and refresh behavior per [02_viewing_loads_en.md](./02_viewing_loads_en.md).  
5. **Sign a BOL**: complete a dry run on a test load if available; understand required fields per [03_signing_bols_en.md](./03_signing_bols_en.md).  
6. **Report an issue**: submit a sample in-transit note using [04_reporting_issues_en.md](./04_reporting_issues_en.md); verify dispatch sees it in office UI.

## Week one — proficiency

7. **Accuracy**: zero missed signatures on delivered loads; photos legible.  
8. **Latency**: updates submitted same day (same tour), not batch-uploaded days later.  
9. **Escalation**: know radio/desk phone path when the app is unavailable—still move the freight safely.

## Compliance reminders

- Never share OTP codes or session links; dispatch resets access if a device is lost.  
- Log out on shared tablets at fuel desks or truck stops.
