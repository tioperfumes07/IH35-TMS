# Runbook — Driver Onboarding Checklist

**Owner:** Safety / HR / Dispatch
**Frequency:** Per new driver
**Systems:** IH35 TMS (Drivers, Safety, Driver PWA, Samsara)

Bring a new driver from applicant to first load, DOT-compliant and system-ready.

## Pre-hire

1. **MVR check** (Motor Vehicle Record) pulled and reviewed.
2. **Drug test** scheduled and passed (pre-employment).
3. **Road test** completed and documented.

## Hire

4. **CDL verification** — class/endorsements valid and not expired.
5. **DOT physical / medical card** on file with expiration tracked.
6. **Hazmat endorsement** (if applicable) verified.
7. Create the driver record in **Drivers** with all profile + compliance fields.
8. Complete the **DQF (Driver Qualification File)** documents via the Safety
   onboarding wizard (identity, CDL, medical card, DQF docs, signatures, I-9).

## System setup

9. **PWA install** — driver installs the Driver app and logs in via invite.
10. **Samsara assignment** — pair the driver to a vehicle/ELD.
11. **Payroll setup** — pay basis, deduction types, escrow, W-9 (contractor) or
    W-4 (employee).

## Training

12. **Load board** — how to view and accept assignments.
13. **Settlement view** — how to read settlements and submit disputes.
14. **Document upload** — POD/BOL and fuel-receipt capture.

## First load

15. **Assign the first load** and confirm the driver can see it in the PWA.

## Verification

- [ ] MVR, drug test, road test complete.
- [ ] CDL, DOT physical, Hazmat (if needed) verified + expirations tracked.
- [ ] DQF complete; driver record created.
- [ ] PWA installed; Samsara paired; payroll configured.
- [ ] Training completed; first load assigned.

## Escalation

Any failed pre-hire screen (MVR/drug/road test) stops onboarding — escalate to
Safety/Owner before proceeding.
