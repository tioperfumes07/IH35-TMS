═══ ITEM 2 — BACKEND SERVICES ═══

apps/backend/src/safety/driver-scheduler.service.ts:

Driver-facing operations:
  listMyLeaveRequests(driver_id, filter)
  createLeaveRequest({driver_id, leave_type, start_date, end_date, 
                      reason, documentation_attachment_id?, 
                      suggested_cover_driver_id?})
    - Validates leave_type valid
    - Validates dates valid (end >= start)
    - Validates advance notice per leave policy
    - Validates no overlap with existing approved leave
    - Validates within annual balance (returns warning if approaching limit)
    - Documentation required for sick > 1 day
    - Auto-generates request_number
    - Status = 'pending_review'
    - Notifies office (PWA + email)
  
  cancelMyLeaveRequest(request_id) — only if status='pending_review'
  
  getMySchedule(driver_id, start_date, end_date)
    Returns calendar data for this driver only

Office-facing operations:
  listPendingRequests(filters: operating_company, manager, urgency)
  getLeaveRequest(id) — full detail with audit log
  
  reviewLeaveRequest(request_id, {
    action: 'approve' | 'approve_modified' | 'deny' | 'defer',
    approved_start_date?, approved_end_date?, modification_reason?,
    denied_reason?
  })
    - For approve: creates rows in driver_leave_days table
    - Updates driver_leave_balances (deduct from appropriate bucket)
    - Notifies driver: PWA + WhatsApp + Email
    - Audit event
  
  getFleetSchedule(operating_company_id, start_date, end_date, filters)
    Returns full calendar grid data:
      - All active drivers + their assigned units
      - Per-day status for each driver
      - Temp assignments
      - Vacant units
      - Pending requests visible as 'pending'
    Optimized query — fetches in 1-2 SQL calls, formats in service
  
  assignTempCover({primary_driver_id, cover_driver_id, unit_id, 
                   start_date, end_date, related_leave_request_id?})
  
  cancelTempCover(temp_assignment_id, reason)
  
  getLeaveBalance(driver_id, year)
  getLeavePolicy(operating_company_id)
  updateLeavePolicy(operating_company_id, updates) — Owner+Admin only

═══ ITEM 7 — CRONS + NOTIFICATIONS ═══

Cron 1: Advance reminder for upcoming leave
  Daily at 08:00 CT
  Find approved leave starting in 3, 7, or 14 days
  Send PWA + email reminders to:
    - Driver (their leave starts soon)
    - Their manager (driver's leave starts soon, ensure coverage)
    - Dispatch (plan loads around it)

Cron 2: Year rollover
  January 1 annually
  For each driver: create new catalogs.driver_leave_balances row 
    for new year with policy defaults
  Apply carryover from prior year if policy allows
  Notify drivers of new annual balance

Cron 3: Pending request escalation
  Daily at 09:00 CT
  Find requests pending review > 5 days
  Escalate to Owner + Administrator via PWA notification + email
